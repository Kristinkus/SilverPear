# SilverPear (Интернет-магазин парфюмерии и косметики)

## Название темы
Веб-приложение для управления списком товаров (парфюмерия и косметика) в интернет-магазине.

## Описание проекта
Данный проект представляет собой RESTful веб-приложение, разработанное с использованием Spring Boot. Оно позволяет выполнять базовые CRUD-операции (создание, чтение, обновление, удаление) для сущности "Товар" (`Product`), а также предоставляет расширенный поиск по различным критериям.

В качестве хранилища данных используется **In-Memory репозиторий**, что позволяет запускать приложение без настройки внешней базы данных. Приложение построено с соблюдением многослойной архитектуры (контроллер, сервис, репозиторий) и использует DTO (`UserDto`) для передачи данных между слоями, скрывая внутреннюю структуру сущности и добавляя вычисляемое поле (`salePrice`).

## Выполняемые функции
Приложение предоставляет REST API для управления товарами:

**1. Получение списка всех товаров**
- **Метод:** `GET`
- **Эндпоинт:** `/shop/api/products`
- **Описание:** Возвращает список всех товаров, хранящихся в "базе данных", преобразованных в DTO.

**2. Получение товара по ID**
- **Метод:** `GET`
- **Эндпоинт:** `/shop/api/products/{id}`
- **Описание:** Возвращает детальную информацию о товаре по его уникальному идентификатору в формате DTO.

**3. Добавление нового товара**
- **Метод:** `POST`
- **Эндпоинт:** `/shop/api/products`
- **Описание:** Принимает JSON с данными нового товара (в формате DTO), сохраняет его и возвращает созданную запись.

**4. Поиск товаров по параметрам**
- **Метод:** `GET`
- **Эндпоинт:** `/shop/api/products/search`
- **Описание:** Выполняет поиск товаров по необязательным параметрам запроса: `name` (название), `brand` (бренд), `category` (категория). Поиск осуществляется по частичному совпадению без учета регистра.

**5. (Дополнительно) Демонстрационные endpoints**
- **Метод:** `GET`
- **Эндпоинт:** `/shop/products`
- **Описание:** Простой endpoint для демонстрации, возвращает список строк (не связан с основной логикой товаров).
- **Метод:** `GET`
- **Эндпоинт:** `/shop/details`
- **Описание:** Демонстрирует работу с параметрами запроса (`@RequestParam`).

## Стек технологий
*   **Язык:** Java 17
*   **Фреймворк:** Spring Boot 4.0.2
*   **Сборщик:** Maven
*   **Хранилище данных:** In-Memory (эмуляция репозитория)
*   **Основные зависимости:**
    *   `spring-boot-starter-web` (для создания REST API)
    *   `lombok` (для упрощения кода)
    *   `modelmapper` (в проекте не используется, но добавлен как зависимость)

## Структура проекта (Архитектура)
Проект следует классической многослойной архитектуре:
*   **Контроллер (`ProductController`):** Принимает HTTP-запросы, обрабатывает пути и параметры, возвращает ответы клиенту.
*   **Сервис (`ProductService`):** Содержит бизнес-логику приложения, управляет транзакциями и взаимодействует с репозиторием.
*   **Репозиторий (`ProductRepository`):** Отвечает за хранение и извлечение данных. В текущей реализации это эмуляция базы данных в памяти (`ArrayList`).
*   **Сущность (`Product`):** Объект предметной области, представляющий товар.
*   **DTO (`UserDto`):** Объект для передачи данных между сервером и клиентом. Скрывает лишние поля (например, `purchasePrice`) и добавляет вычисляемые (`salePrice`).
*   **Маппер (`UserMapper`):** Компонент для преобразования между сущностью `Product` и DTO `UserDto`.
